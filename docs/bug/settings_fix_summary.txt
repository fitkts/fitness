문제 원인:
1. Settings.tsx의 useEffect 훅에서 showToast를 의존성 배열에 포함시켜 무한 루프 발생
2. 설정 로드 시마다 토스트 메시지가 표시되어 상태 업데이트가 계속 발생

해결 방법:
1. useEffect의 의존성 배열에서 showToast 제거
2. 초기 로딩 시에만 토스트 메시지가 표시되도록 조건 추가
3. isLoading 상태를 체크하여 중복 토스트 메시지 방지

결과:
- 설정 페이지 로드 시 한 번만 설정을 불러오도록 수정
- 불필요한 상태 업데이트와 리렌더링 방지
- 사용자 경험 개선
