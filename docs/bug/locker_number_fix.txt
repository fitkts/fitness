락커 번호 문제 해결 내용

문제 원인:
1. 락커 번호가 '002'와 같이 앞에 0이 있는 형식으로 저장됨
2. 정렬 시 CAST(l.number AS INTEGER)를 사용하여 문자열 비교가 아닌 숫자 비교가 이루어짐
3. 중복 체크 시 정규화되지 않은 번호로 비교하여 오류 발생

해결 방안:
1. 락커 번호 정규화 함수 추가
   - 앞의 0을 제거하고 숫자만 남기는 normalizeLockerNumber 함수 구현
   - 락커 추가 시 정규화된 번호로 저장

2. 중복 체크 로직 개선
   - 정규화된 번호로 중복 체크 수행
   - CAST(REPLACE(number, "0", "") AS INTEGER) 사용하여 숫자 비교

3. 정렬 방식 수정
   - ORDER BY CAST(REPLACE(l.number, "0", "") AS INTEGER) 사용
   - 앞의 0을 제거하고 숫자로 정렬

4. 검증 로직 개선
   - validateLockerNumber 함수에서 정규화된 번호로 검증
   - 1-999 범위 체크 시 정규화된 번호 사용

개선 효과:
1. 락커 번호 중복 문제 해결
2. 일관된 정렬 순서 제공
3. 사용자 입력 오류 방지
4. 데이터 일관성 향상
